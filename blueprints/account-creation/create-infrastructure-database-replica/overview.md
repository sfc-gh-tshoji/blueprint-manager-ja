このステップでは、セカンダリレプリケーショングループとレプリカデータベースを作成し、このアカウントが FinOps タグ、ネットワークルール、コストレポートビューなどの集中管理されたガバナンスオブジェクトにアクセスできるようにします。共有データベースとは異なり、レプリケートされたデータベースは完全に使用できる**ローカルコピー**を作成します — ローカルオブジェクトへのタグ適用も含みます。

**アカウントコンテキスト:** このステップは**新しく作成したアカウント**（組織アカウントではなく）から実行します。

## **なぜこれが重要か？**

インフラレプリカを作成することで、このアカウントに以下が提供されます:
- **ローカルタグ**: タグがウェアハウス、データベース、その他のリソースに設定できるローカルオブジェクトになる
- **ローカルネットワークルール**: ネットワークルールをローカルネットワークポリシーで参照できる
- **コストレポートビュー**: アカウント間のコスト配分への可視性
- **自動同期**: 組織アカウントの変更が自動的にレプリケートされる

**なぜ共有ではなくレプリケーションか？**
共有データベースは読み取り専用です — 共有タグを表示できますが、ローカルオブジェクトに**適用することはできません**。レプリケートされたデータベースはローカルに作成されたかのように機能するローカルコピーを作成し、以下を可能にします:
- レプリケートされたタグを使用した `ALTER WAREHOUSE SET TAG ...` ✅
- レプリケートされたネットワークルールを参照するネットワークポリシー ✅
- すべてのアカウント間での完全なガバナンス機能 ✅

## **前提条件**

- アカウントの正常な作成（アカウント作成ステップ）
- ACCOUNTADMIN として新しいアカウントにログイン済み
- このアカウントがレプリケーショングループの ALLOWED_ACCOUNTS に追加済み（組織アカウントステップで実行）

## **主要な概念**

**レプリケーショングループ**
アカウント間でユニットとしてレプリケートされるオブジェクトの定義済みコレクション。*プライマリ*レプリケーショングループはソースアカウント（組織アカウント）に存在し、*セカンダリ*レプリケーショングループはターゲットアカウント（このアカウント）に存在します。

**共有 vs レプリケーションの主な違い**

| 機能 | 共有データベース | レプリケートされたデータベース |
|------|--------------|--------------------------|
| タグの表示 | ✅ はい | ✅ はい |
| ローカルオブジェクトへのタグ適用 | ❌ いいえ | ✅ はい |
| ポリシーでのネットワークルール参照 | ❌ いいえ | ✅ はい |
| ビューのクエリ | ✅ はい | ✅ はい |
| 自動同期 | ✅ 即時 | ✅ スケジュール通り |

### 設定の質問

#### インフラデータベースレプリケーショングループの名前は何にしますか？（`infrastructure_replication_group`: text）
インフラデータベースを他のアカウントに同期するレプリケーショングループオブジェクトの名前を選択します。
**デフォルトの推奨:** infrastructure_replication_group

#### Snowflake 組織名は何ですか？（`snowflake_org_name`: text）
Snowflake 組織名は URL の最初の部分です。

#### 組織アカウントの名前は何にしますか？（`org_account_name`: text）
**推奨名:** ORG

#### プラットフォームデータベースの名前は何にしますか？（`platform_database_name`: text）
インフラデータベースの名前（例: PLAT\_INFRA）。

#### このアカウントに使用する名前は何ですか？（`new_account_name`: text）
新しいアカウントの名前を確認します。

#### ガバナンススキーマの名前は何にしますか？（`governance_name`: text）
**推奨名:** GOVERNANCE

#### どのアカウント戦略を実装しますか？（`account_strategy`: multi-select）
**オプション:**
- Single Account
- Multi-Account (Environment-based)
- Multi-Account (Domain-based)
- Multi-Account (Domain + Environment)
