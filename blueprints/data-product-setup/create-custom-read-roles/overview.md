カスタム読み取りロールにより、標準のデータ製品ロールを超えたきめ細かいアクセス制御が可能になります。すべてのゾーンとスキーマへのアクセスを提供する完全な READ ロールを付与する代わりに、特定のデータサブセットへのアクセスを提供するカスタムロールを作成できます。

カスタム読み取りロールのユースケース:
- **外部パートナー**: 特定のキュレーション済みデータセットのみへのアクセス
- **ビジネスユニット**: 関連するドメインのみへのアクセス
- **コンプライアンス**: データ分類に基づいた制限付きアクセス
- **アプリケーション**: 特定のサービスのスコープ付きアクセス

各カスタムロールはスキーマレベルのデータベースロールで構成され、どのスキーマにアクセスできるかを精密に制御します。

**アカウントコンテキスト:** RBAC ロールを使用してターゲットアカウントでこの SQL を実行します。

## なぜこれが重要か？

カスタム読み取りロールにより以下が可能になります:
- **最小権限**: 必要なアクセスのみを付与
- **データ共有**: 安全な外部データアクセス
- **コンプライアンス**: 規制要件を満たす
- **柔軟性**: ユースケースごとにロールを作成

## 前提条件

- データベースとスキーマの作成済み（ステップ 2.2、2.3）
- ロール階層の接続済み（ステップ 3.3）

## 主要な概念

**カスタムロールの構成**
カスタムロールはスキーマレベルのデータベースロールを付与することで構築されます:
```
カスタムロール → SC_R_<schema> → DB_R → READ
```

**命名規則**
```
<prefix>_CUSTOM_<name>
```

**所有権**
- カスタムロールは RBAC ロールによって所有される
- RBAC は ADMIN の関与なしに付与を委任可能

**付与マトリックス**
| アクセスレベル | 付与するデータベースロール |
|--------------|--------------------------|
| スキーマ全体の読み取り | SC_R_<schema> |
| 複数スキーマ | SC_R_<schema1>、SC_R_<schema2> |
| ゾーン全体の読み取り | DB_R |
| 複数ゾーン | 各データベースの DB_R |

**追加情報:**
* [ロールベースアクセス制御](https://docs.snowflake.com/en/user-guide/security-access-control-overview)
* [データベースロール](https://docs.snowflake.com/en/user-guide/security-access-control-overview#database-roles)


### 設定の質問

#### このデータ製品の名前は何ですか？（`data_product_name`: text）
データ製品の説明的な名前を提供します。

#### どのアカウント戦略を実装しますか？（`account_strategy`: multi-select）
**オプション:**
- Single Account
- Multi-Account (Environment-based)
- Multi-Account (Domain-based)
- Multi-Account (Domain + Environment)

#### このデータ製品はどのアカウントにデプロイされますか？（`target_account_name`: text）
Snowflake アカウントの正確な名前を入力します。

#### このデータ製品が属するドメインはどれですか？（`data_product_domain`: multi-select）
プラットフォームファウンデーションで定義されたビジネスドメインを選択します。

#### このデータ製品がデプロイされる環境はどれですか？（`data_product_environment`: multi-select）
デプロイメントの SDLC 環境を選択します。

#### きめ細かいアクセス制御のためのカスタム読み取りロールを定義してください。（`custom_read_roles`: object-list）
外部パートナー、特定のチーム、または特定のスキーマへの限定的な読み取りアクセスが必要なアプリケーション向けのカスタムロールを定義します。

**カスタムロールを使用する場合:**
- データのサブセットが必要な外部パートナー
- 専用ロールが必要な BI ツール
- コンプライアンスによるアクセス分離

**設定例:**
| ロール名 | 説明 | スキーマアクセス |
|---------|------|----------------|
| PARTNER_ACME | Acme Corp データアクセス | CONSUMPTION.PARTNER_DATA |
| BI_TEAM | Tableau/PowerBI アクセス | CONSUMPTION.REPORTING |
| ML_TEAM | ML フィーチャーアクセス | CURATED.FEATURES |

**スキーマアクセスの形式:**
複数のスキーマはカンマ区切りで `ZONE.SCHEMA` 形式を使用します。

**注:** これらのロールは読み取りアクセスのみを受け取ります。書き込みアクセスには、標準の機能ロール（WRITE、CREATE）を使用します。
