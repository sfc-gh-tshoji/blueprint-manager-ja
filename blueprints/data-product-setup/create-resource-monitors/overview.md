リソースモニターはクレジット消費を追跡し、しきい値を超えたときにアクションを取ることで、データ製品のコスト制御を提供します。このステップでは以下を含むリソースモニターを作成します:

1. **クレジットクォータ** — 期間あたりに許可される最大クレジット
2. **モニタリング頻度** — クォータのリセット頻度（日次、週次、月次など）
3. **通知しきい値** — 消費追跡のためのアラートレベル
4. **アクショントリガー** — しきい値を超えたときの自動アクション

リソースモニターは、特にマルチテナント環境での Snowflake コストを制御するために不可欠です。

**アカウントコンテキスト:** リソースモニターを作成するには ACCOUNTADMIN でこの SQL を実行します。

## なぜこれが重要か？

リソースモニターにより以下が可能になります:
- **コスト制御**: ランナウェイなコンピューティング支出を防ぐ
- **予算コンプライアンス**: 割り当てられたクレジット内に収める
- **可視性**: 消費パターンを追跡する
- **自動化**: 予算が枯渇したときの自動一時停止

## 前提条件

- ウェアハウスの作成済み（ステップ 3.1）
- ACCOUNTADMIN ロールへのアクセス

## 主要な概念

**モニター頻度オプション**

| 頻度 | 説明 |
|------|------|
| DAILY | クォータが毎日リセット |
| WEEKLY | クォータが毎週リセット |
| MONTHLY | クォータが毎月リセット（推奨） |
| QUARTERLY | クォータが四半期ごとにリセット |
| YEARLY | クォータが毎年リセット |
| NEVER | クォータがリセットされない |

**しきい値アクション**

| アクション | 効果 |
|-----------|------|
| NOTIFY | 通知のみ送信 |
| SUSPEND | 現在のクエリ完了後にウェアハウスを一時停止 |
| SUSPEND_IMMEDIATE | ウェアハウスを即時一時停止 |

**推奨しきい値**
- 50% → 通知（事前通知）
- 75% → 通知（上限に近づいている）
- 90% → 通知（緊急）
- 100% → 一時停止（過剰支出を防ぐ）

### 設定の質問

#### このデータ製品の名前は何ですか？（`data_product_name`: text）
データ製品の説明的な名前を提供します。

#### どのアカウント戦略を実装しますか？（`account_strategy`: multi-select）
**オプション:**
- Single Account
- Multi-Account (Environment-based)
- Multi-Account (Domain-based)
- Multi-Account (Domain + Environment)

#### このデータ製品はどのアカウントにデプロイされますか？（`target_account_name`: text）
Snowflake アカウントの正確な名前を入力します。

#### このデータ製品が属するドメインはどれですか？（`data_product_domain`: multi-select）
プラットフォームファウンデーションで定義されたビジネスドメインを選択します。

#### このデータ製品がデプロイされる環境はどれですか？（`data_product_environment`: multi-select）
デプロイメントの SDLC 環境を選択します。

#### リソースモニターのクレジット上限は何クレジットにしますか？（`resource_monitor_credits`: text）
このデータ製品のウェアハウスの最大クレジット消費量を設定します。

**推定方法:**
- X-Small ウェアハウス ≈ 1 クレジット/時間
- Small ウェアハウス ≈ 2 クレジット/時間
- Medium ウェアハウス ≈ 4 クレジット/時間
- Large ウェアハウス ≈ 8 クレジット/時間

**計算例:**
- 軽い使用（1日8時間、Small）: 8 × 2 × 22 = 352 クレジット/月
- 中程度の使用（1日12時間、Medium）: 12 × 4 × 22 = 1,056 クレジット/月
- 重い使用（24時間365日、Large）: 24 × 8 × 30 = 5,760 クレジット/月

**推奨:** 控えめに始めてください。後で上限を増やすことができます。

**例:**
- `500` — 小規模データ製品
- `1000` — 中程度のワークロード
- `5000` — 重い ETL と分析

#### リソースモニターのクレジットクォータはどのくらいの頻度でリセットしますか？（`resource_monitor_frequency`: multi-select）
クレジットクォータのリセット頻度を選択します。

**推奨:**
- **Daily**: 厳密なコスト制御、実験的なワークロード
- **Weekly**: 開発環境
- **Monthly**: ほとんどの本番データ製品（推奨）
- **Quarterly/Yearly**: 安定した予測可能なワークロード
- **Never**: ライフタイムの上限を設定

**オプション:**
- Daily
- Weekly
- Monthly
- Quarterly
- Yearly
- Never（ライフタイム上限）
